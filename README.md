# Subby

> Know where your money flows

A beautiful, local-first desktop subscription tracker built with Tauri 2, React, and SQLite.

## Features

- **Dashboard** - Monthly/yearly spending stats, category breakdown donut chart, spending trend chart
- **Subscription Management** - Add, edit, delete, pause subscriptions with grid/list views
- **Payment Calendar** - Visual calendar with payment indicators, mark as paid/skipped
- **Category Organization** - 9 default categories + custom categories with color coding
- **Data Management** - Export/import JSON backups for full data portability
- **Themes** - Dark/light/system theme support
- **Multi-currency** - Support for USD, EUR, GBP, and more
- **100% Offline** - All data stored locally in SQLite, no account required

## Tech Stack

| Layer        | Technology                       |
| ------------ | -------------------------------- |
| **Runtime**  | Tauri 2.x (Rust backend)         |
| **Frontend** | React 19 + TypeScript + Vite     |
| **Styling**  | Tailwind CSS 4 + shadcn/ui       |
| **State**    | Zustand                          |
| **Database** | SQLite via tauri-plugin-sql      |
| **Forms**    | React Hook Form + Zod validation |
| **Charts**   | Custom SVG components            |

## Getting Started

### Prerequisites

- Node.js 18+
- pnpm 8+
- Rust (via [rustup](https://rustup.rs/))

**Linux (Debian/Ubuntu)**:

```bash
sudo apt install libgtk-3-dev libwebkit2gtk-4.1-dev libappindicator3-dev librsvg2-dev patchelf
```

**macOS**: Xcode Command Line Tools  
**Windows**: WebView2, Visual Studio Build Tools

### Installation

```bash
git clone https://github.com/newstella/subby.git
cd subby
pnpm install
pnpm tauri dev
```

### Build for Production

```bash
pnpm tauri build
```

Binaries will be in `src-tauri/target/release/bundle/`.

### Releases (Windows/macOS/Linux)

Release installers are generated by GitHub Actions when a `v*` tag is pushed. Artifacts are attached to the GitHub Release for each OS.

1. Update versions in `src-tauri/tauri.conf.json` and `package.json`.
2. Create and push a tag, for example `git tag v0.1.1` then `git push --tags`.

For signed production releases, configure platform signing secrets in GitHub Actions (Apple Developer ID and Windows code signing certificate). Unsigned builds will still run but may show OS warnings.

## Data Storage

Your data stays on your device:

| Platform | Location                               |
| -------- | -------------------------------------- |
| Linux    | `~/.local/share/subby/`                |
| macOS    | `~/Library/Application Support/subby/` |
| Windows  | `%APPDATA%/subby/`                     |

